# Planned Submodules

This page provides an overview of all planned Git submodules for different research domains.

## Overview

Each submodule is an independent Git repository focused on a specific engineering domain. This separation provides focused version control, independent development cycles, and modular learning progression.

## Submodules

### 1. embedded-linux-labs [TODO]

Personal embedded Linux learning labs and experiments

**Focus**: Toolchain setup, kernel compilation, board bring-up, device drivers

### 2. yocto-projects [TODO]

Yocto Project build system research and custom distributions

**Focus**: BitBake, custom layers, image creation, SDK generation

### 3. linux-kernel-debugging [TODO]

Linux kernel debugging techniques and tools

**Focus**: Module development, tracing, profiling, crash analysis

### 4. device-drivers [TODO]

Linux device driver development

**Focus**: Character devices, platform drivers, device tree integration

### 5. realtime-systems [TODO]

Real-time Linux (PREEMPT-RT, Xenomai)

**Focus**: Latency analysis, real-time optimization, deterministic systems

### 6. build-systems [TODO]

CMake, Meson, Autotools research

**Focus**: Modern build systems, cross-compilation, dependency management

### 7. devops-homelab [TODO]

Infrastructure automation and CI/CD pipeline

**Focus**: Terraform, Ansible, Kubernetes, monitoring, deployment

### 8. hardware-platforms [TODO]

SBC and microcontroller platform documentation

**Focus**: Raspberry Pi, BeaglePlay, ESP32, Arduino, platform specs

### 9. network-protocols [TODO]

Protocol implementation and analysis

**Focus**: Socket programming, custom protocols, packet analysis

### 10. system-performance [TODO]

Performance analysis and optimization

**Focus**: Profiling, cache optimization, benchmarking

## Creation Priority

Recommended order for creating submodules:

1. **hardware-platforms** - Foundation for all hardware work
2. **embedded-linux-labs** - Core embedded Linux skills
3. **build-systems** - Understanding compilation processes
4. **device-drivers** - Hardware interfacing
5. **linux-kernel-debugging** - Debugging capabilities
6. **yocto-projects** - Distribution building
7. **devops-homelab** - Automation infrastructure
8-10. Remaining domains as needed
